사용법 · 채널별 규칙 (펼치기)
  • 흐름: 매핑 불러오기 → 파일 끌어넣기 → 채널 자동 식별 확인 → [변환 · 다운로드] → 받은 파일을 기존 e-포스트 양식으로 업로드 (업로드 절차 무변경)
  • 스스: 첫 행(안내문) 자동 삭제 · D(상품명) ← T(옵션관리코드) 매핑 · E(옵션정보) 공란
  • 쿠팡(판배): K(등록상품명) ← O(옵션ID) 매핑 · L·M·P 공란
  • 선물하기: G(상품명) ← F(채널상품번호)+H(옵션) 매핑 · H 공란 · F 숫자화 — 톡스토어는 "미매핑"으로 뜸 → 현행대로 수기. ★ 주문 파일을 넣어야 함 (코드표 파일 아님)
  • 전 채널 검증 완료(7/8): 카페24는 6개월 실주문 932행 대조로 검증(비쿠팡 121/125 변환·오변환 0). 카페24 파일의 쿠팡행 미매핑은 정상 — 쿠팡은 WING 파일로 처리하므로 무시(평소대로)
  • 주소 오변환 수정(7/8): SSG·카페24 파일을 원문 그대로 읽음 — "방답길 36-9" 같은 주소가 날짜로 바뀌던 버그 해결
  • 역등록(3번): 채널 파일들을 2)에 넣은 채로 반환 파일을 3)에 넣고 버튼 — 스스(J송장·I택배사)·선물하기(L송장·K006)·쿠팡(E송장·D우체국) 자동 채움, 카페24는 L송장 채움 + 배송중처리.csv 자동 생성(그대로 사이트 등록), 우체국쇼핑·SSG는 대조표. 주문번호는 e-포스트 접수 양식에 매핑된 접수분부터 돌아옴(7/8 수정)
  • 미매핑 행: 원본 그대로 두고 목록으로 표시 → 마스터 탭에 기입 보완 후 다시 변환하면 됨(재변환 안전)
